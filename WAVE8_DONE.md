# Wave 8 — безопасный перенос прогресса + browser E2E

## Что сделано

### 1. Резервная копия без облака
- В `engine10.js` добавлен безопасный перенос текущего класса без npoint.io:
  - экспорт в `.json`
  - экспорт в длинный `код переноса`
  - импорт из файла
  - импорт из кода
- В копию входят:
  - имя и код игрока
  - приватный режим
  - progress / daily / activity / streak
  - даты диагностик
  - журнал ошибок
  - локальные рекорды `Молнии`
  - фильтр `Всё вперемешку`
- На welcome-экране (`pickName`) появился путь восстановления **из файла / кода переноса**.
- На всех `grade*_v2.html` добавлена кнопка **`💾 Резервная копия`** рядом с отчётом для родителя.
- В справке обновлено описание блока `☁️ Облако и код`.

### 2. Browser E2E smoke
- Добавлен `browser_e2e_smoke.py` на Playwright + Chromium.
- Тесты идут не по Node-имитации, а в headless-браузере.
- Покрыты сценарии:
  - `grade1_v2.html`: регистрация → запуск темы → ответ → сохранение состояния → export/import backup
  - `grade2_v2.html`: то же + проверка вариативности `genMeasure`
  - `grade5_v2.html`: базовый flow по обычному классу
  - `grade10_v2.html`: flow + наличие новых английских тем
  - `diagnostic.html`: `mathall`, ответ, `skip`, продвижение по адаптивной цепочке
- Автоматически генерируется `BROWSER_E2E_REPORT.md`.

### 3. Контент по замечаниям Claude / DeepSeek

- Усилена вариативность для `grade1`:
  - `genCount`
  - `genSub`
  - `genShapes`
  - `genLetters`
- Усилена вариативность для `grade2`:
  - `genMeasure`
  - `genBezud`

  - `genAnimals`
- Для `grade10` в английском добавлены темы:
  - `Фразовые глаголы` (`phrasal`)
  - `Написание эссе` (`essay`)
  - с теорией и отдельными генераторами заданий.

### 4. PWA / release
- `sw.js` обновлён до `trainer-v6`.
- В precache добавлен `wave8_boosters.js`.
- `validate_release.py` теперь прогоняет и browser E2E.

## Проверки
- `node --check engine10.js` ✓
- `node --check wave8_boosters.js` ✓
- `python browser_e2e_smoke.py` ✓
- `python validate_release.py` ✓

## Важная честная оговорка
- В этой волне сделан **безопасный export/import через файл и код переноса**.
- Полноценный **QR-скан/генерация QR внутри браузера** пока **не реализован**.
- То есть быстрый путь из плана закрыт по сути, но не в QR-формате, а через надёжный файл/код без облака.

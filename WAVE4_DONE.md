# Wave 4 — UX + mesh audit + stricter validation

## Что сделано

### 1) Поиск по предметам и темам
- Во все `grade*_v2.html` добавлены слоты поиска:
  - `#main-search-slot` на главном экране
  - `#topic-search-slot` внутри предмета
- В `engine10.js` добавлен поиск:
  - по предметам и темам на главном экране
  - по темам внутри выбранного предмета
- При поиске показываются:
  - число найденных предметов/тем
  - empty-state, если совпадений нет
  - chip с числом совпавших тем, если совпал не предмет, а темы внутри него

### 2) Слабые темы стали рабочим инструментом
- `showJournal()` переведён на безопасные и темизированные модалки
- Для каждой слабой темы добавлены быстрые действия:
  - `📖` открыть шпаргалку
  - `✏️` сразу начать тренировку по теме
- `startWeakTraining()` теперь:
  - усиливает веса реально слабых тем
  - автоматически ограничивает mix соответствующими предметами
  - сохраняет этот фильтр

### 3) Валидация дат диагностик
- `showDateEditor()` переведён на `var(--card) / var(--text) / var(--border)`
- `applyDates()` теперь валидирует:
  - заполнены ли и день, и месяц
  - корректна ли дата по календарю
  - нормализует день (`05` -> `5`)
- Некорректные поля подсвечиваются, пользователю показывается список ошибок

### 4) Фильтр микса теперь запоминается
- Добавлены `loadMixFilter()` / `saveMixFilter()`
- `showMixFilter()` открывается с последним выбранным набором предметов
- `launchMix()` сохраняет фильтр
- `startWeakTraining()` тоже использует и сохраняет фильтр

### 5) Индекс и актуальные числа
- В `index.html` обновлены счётчики тем:
  - 1 класс: `13 тем`
  - 2 класс: `12 тем`

### 6) Аудит сетки предметов и тем стал глубже
- `curriculum_audit.js` переписан
- Теперь он строит:
  - сводку по классам
  - список предметов по каждому классу
  - полную сетку предметов `1–11`
  - список повторяющихся `subject/topic`
  - список «тонких» блоков по темам
- Генерируются два артефакта:
  - `CURRICULUM_AUDIT.md`
  - `CURRICULUM_MESH.json`

### 7) Усилен контроль релиза
- `runtime_smoke_check.js` теперь проверяет не только инициализацию сетки, но и наличие новых search UI
- `validate_release.py` теперь запускает ещё и `curriculum_audit.js`

## Итог
Wave 4 закрыл следующий слой после стабилизации ядра:
- навигация по большому объёму тем стала быстрее
- слабые темы превратились в точку входа обратно в обучение
- даты диагностик больше не принимают мусор
- предметная сетка теперь контролируется не руками, а машинным аудитом

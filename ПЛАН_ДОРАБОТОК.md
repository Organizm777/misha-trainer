## UX / ПРОДУКТ

### 4. Онбординг для младших классов
Grade1-2 теперь получили движок grade10 с режимом «Молния», диагностикой, датами. Для первоклассников это перегруз. Варианты:
- Скрывать продвинутые кнопки для 1-4 классов через `if(GRADE_NUM<=4)`
- Или оставить, но упростить текст подсказок

### 5. Тактильная обратная связь
vibrate() работает не везде. Добавить визуальный feedback (flash-анимация на карточке ответа) как fallback.

### 6. Прогресс между сессиями
Dashboard показывает общий прогресс, но нет графика «задач в день за неделю/месяц». Можно добавить в dashboard.html на основе trainer_dates_{N}.

### 7. Экспорт результатов
Родителю неудобно скриншотить. Добавить кнопку «Скачать отчёт» в dashboard — генерация PDF/PNG с результатами.

## ТЕХНИЧЕСКОЕ

### 8. npoint.io race condition
Last-write-wins при одновременной работе. Варианты:
- Перейти на Firebase Realtime DB (бесплатный tier)
- Или Supabase (postgres + REST)
- Или оставить npoint но хранить одного игрока в отдельном бине

### 9. Оптимизация grade10_v2 (720 КБ)
458 КБ — это QBANK с вопросами. Можно:
- Вынести вопросы в отдельный JSON и подгружать лениво
- Или разделить grade10 по предметам (grade10_alg.json, grade10_phy.json)
- Экономия при первой загрузке: грузить только 2-3 активных предмета

### 10. E2E тестирование
Сейчас проверка только синтаксическая. Нужны функциональные тесты:
- Открыть страницу → проверить что предметы отображаются
- Начать квиз → ответить → проверить подсчёт
- Проверить localStorage после ответа
Инструмент: Playwright или Puppeteer.

### 11. Минификация
engine10.js (84 КБ) — читаемый код. После минификации (terser) → ~40 КБ.
С gzip: ~12 КБ. Это ускорит первую загрузку.

### 12. Переименование v2 → основные файлы
Когда v2 протестированы, переименовать:
```
grade{N}_v2.html → grade{N}.html
```
И обновить все ссылки обратно. Или оставить _v2 как постоянное имя.

# wave87i: question repeat guard

## Что меняется

wave87i усиливает `chunk_subject_expansion_wave63_quality` в двух местах:

1. На уровне `QBANK` удаляются не только точные дубли `question+answer`, но и строки с одинаковым нормализованным текстом вопроса. Если у одинакового вопроса совпадает ответ, строка считается дублем; если ответ отличается, строка помечается как конфликт и не попадает в рабочий банк.
2. На уровне `topic.gen()` вводится короткая память последних вопросов. Генератор делает несколько повторных попыток, если новая выдача совпала с одним из последних промптов этой темы.

## Зачем

Это закрывает практическую часть B7: ученик реже видит мгновенные повторы в коротких сериях, а новые live-bank слои получают базовую защиту от дублей и конфликтных строк.
Дополнительно волна подчистила старые малые банки, где разные задания имели один и тот же общий stem вроде «Как правильно?» или `Choose the correct spelling.`. Теперь эти темы тоже дают различимые промпты, а guard учитывает размер банка не только по live-bank метаданным, но и по legacy `rows`/coverage-count.

## Как проверять

```bash
node tools/audit_question_repeat_guard_wave87i.mjs
node tools/validate_questions.js
```

Первый скрипт проверяет live-bank темы на немедленные повторы и низкое разнообразие. Второй остаётся общим регрессионным валидатором и теперь дополнительно выводит метрики repeat-guard слоя.

# DAILY_QUESTION_wave88a

## Что добавлено

`wave88a` закрывает пункт дорожной карты `#52`: на главной странице появился отдельный блок **«Задание дня»**.

Новые source assets:

- `assets/_src/js/chunk_roadmap_wave88a_daily_question.js`
- `assets/_src/css/wave88a_daily_question.css`

Карточка вставляется прямо на `index.html`, то есть ученик видит вопрос ещё до входа в конкретный класс и может ответить без перехода в тренажёр.

## Как работает

- вопрос дня выбирается детерминированно по локальной календарной дате;
- состояние ответа хранится в `localStorage` с отдельным ключом на каждый день;
- после ответа сразу показывается разбор и ссылка в соответствующий класс;
- на следующий календарный день карточка автоматически переключается на новый вопрос.

Текущий пул — **16 стартовых вопросов** по разным классам и предметам:

- алгебра;
- русский язык;
- физика;
- химия;
- информатика;
- история;
- английский язык;
- геометрия;
- биология;
- вероятность;
- литература;
- география;
- обществознание.

## UX-решения

Карточка нарочно сделана самостоятельной и лёгкой:

- отдельный бейдж `Задание дня`;
- один вопрос и четыре ответа;
- подсказка до ответа;
- статус `✓` / `✗` и короткий разбор после ответа;
- CTA-кнопка `Открыть N класс`.

Это не заменяет обычную тренировку, а служит коротким ежедневным входом в продукт.

## Почему отдельный chunk, а не код внутри index

По архитектуре проект уже живёт на внешних hashed assets без inline script/style. Поэтому `wave88a` продолжает тот же подход:

- HTML остаётся статичным;
- логика живёт в отдельном JS-chunk;
- стили вынесены в отдельный CSS asset;
- SW и asset-manifest синхронизируются как у остальных roadmap-пассов.

## Что intentionally не делалось

- не добавлялась серверная синхронизация вопроса дня;
- не считались отдельные награды/XP за этот блок;
- не использовался большой общий банк grade-данных — пока это компактный curated pool для главной страницы;
- не строился новый routing flow: ссылка ведёт в уже существующий grade page.

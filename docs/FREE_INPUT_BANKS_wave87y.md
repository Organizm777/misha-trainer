# FREE_INPUT_BANKS_wave87y

## Что добавлено

`wave87y` продолжает `wave87x`: свободный ввод теперь не только поддерживается рантаймом, но и получает отдельный слой явного контента для 8–11 классов.

Новый source chunk: `assets/_src/js/chunk_subject_expansion_wave87y_free_input_banks.js`.

Он добавляет explicit-bank темы с `inputMode: 'numeric'` в старшие классы:

- 8 класс: алгебра, вероятность, физика, химия;
- 9 класс: алгебра, физика, химия;
- 10 класс: алгебра, вероятность, физика, химия;
- 11 класс: алгебра, вероятность, физика, химия.

Итого: **15 новых numeric-input тем** и **76 explicit rows**.

## Зачем это понадобилось

В `wave87x` numeric/cloze режимы уже работали, но часто включались по эвристике:

- по `___` в тексте вопроса;
- по виду ответа;
- по предмету (`alg|geo|phy|chem|prob`).

Этого достаточно для автоматического покрытия старого контента, но недостаточно для целевого free-input UX. Новые темы в `wave87y`:

- сразу помечены как `inputMode: 'numeric'`;
- имеют `acceptedAnswers` для альтернативных форм записи;
- содержат theory-блок, уже написанный под формат «введи число / дробь / число с единицей».

## Какие форматы ответа поддержаны

### 1. Дроби и десятичная запись

Для вероятностей и части физики ответы допускают оба формата:

- `1/2` ↔ `0,5` / `0.5`
- `1/4` ↔ `0,25` / `0.25`
- `2/5` ↔ `0,4` / `0.4`

Вместе с этим `bundle_grade_runtime_inputs_timing_wave87x` доработан:

- дроби теперь парсятся как числа, а не как «число + странная единица»;
- placeholder/help-текст для numeric input умеет показывать пример вида `1/2 или 0,5`.

### 2. Единицы измерения

Для физики и части химии `acceptedAnswers` включают запись:

- только числом (`4`),
- с пробелом (`4 А`),
- без пробела (`4А`).

То есть ученик может вводить число в привычном виде, не спотыкаясь о формат.

### 3. Проценты

Для массовой доли/процентов поддерживаются:

- `10`
- `10%`
- `10 %`

## Важная архитектурная деталь для grade 10

Grade 10 остаётся lazy-split по предметам (`wave86s`).

Новый wave87y chunk **не вызывает** `__wave86sApplyGrade10Subject(...)` заранее, потому что этот helper помечает предмет как уже загруженный. Если вызвать его слишком рано, можно случайно пропустить основной lazy subject chunk.

Вместо этого wave87y:

- патчит `window.SUBJ` напрямую,
- добавляет новые темы в subject shell,
- не меняет `_wave86sLoaded`/`loadedMap`,
- оставляет существующий lazy-loader grade10 без регрессий.

## Что intentionally не делалось

- не менялся scoring engine;
- не добавлялся отдельный UI-режим для дробей — используется тот же free input;
- не размечались difficulty-tags для всех новых rows;
- не переносились старые эвристические numeric-вопросы в explicit-bank формат массово — `wave87y` добавляет новый управляемый слой, а не переписывает весь старый банк.

# FREE_INPUT_TIMING_wave87x

## Что добавлено

`wave87x` закрывает следующий практический слой дорожной карты без переписывания `engine10`:

- свободный ввод для `cloze`-вопросов и числовых ответов;
- журнал времени ответа по каждому вопросу с показом сводки на экране прогресса;
- стартовые русские темы с пропусками для 8/9/10/11 классов;
- подключение runtime на всех grade-страницах 1–11 классов.

Основной файл: `assets/_src/js/bundle_grade_runtime_inputs_timing_wave87x.js`.

## Как определяется режим ввода

Runtime работает как eager patch поверх уже загруженного `engine10` и `bundle_grade_runtime_core_wave87n`.

### 1. Cloze

Вопрос переводится в режим свободного текстового ввода, если выполняется одно из условий:

- у строки явно стоит `inputMode: 'cloze'` / `inputMode: 'text'`;
- в тексте вопроса есть маркер пропуска `___`.

Это позволяет автоматически подхватить уже существующие английские банки с пропусками, не правя сотни строк контента вручную.

### 2. Numeric

Вопрос переводится в режим числового ввода, если:

- явно указан `inputMode: 'numeric'`;
- ответ выглядит как число / число с единицей измерения;
- вопрос не является code-tracing (`prob.code` пуст);
- задача похожа на математическую/формульную (`prob.isMath` или профильный предмет `alg|geo|phy|chem|prob`).

Нормализация допускает:

- запятую и точку;
- `5 А` и `5А`;
- ввод без единицы измерения, если в эталоне единица есть.

## Как это встраивается в старый renderer

Новый runtime не меняет внутренний scoring engine. Вместо этого он:

1. после `render()` подменяет блок `#opts` на кастомный input UI;
2. при отправке ответа добавляет введённое значение в `prob.options`;
3. вызывает существующий `ans(idx)`;
4. даёт старому движку дальше считать streak, accuracy, журнал ошибок и explanations.

То есть проверка, подсказки, streak и `nextQ()` остаются в одном месте.

## Лог времени ответа

Хранилище: `localStorage['trainer_response_timing_<grade>']`.

Каждая запись содержит компактный sample:

- `ts`
- `ms`
- `qid`
- `grade`
- `mode` (`train` / `rush` / `diag`)
- `subject`
- `tag`
- `correct`
- `usedHelp`
- `inputMode`

Ограничения:

- хранится только последние `360` sample на класс;
- в карточке прогресса анализируются последние `120` ответов обычного тренажёра;
- слишком длинные «зависшие» интервалы (>20 минут) отбрасываются.

## UI

### Экран вопроса

- input + кнопка `Проверить`;
- `Enter` отправляет ответ;
- кнопки `1-4 / A-D` блокируются, если активен режим свободного ввода;
- после ответа показываются:
  - введённое значение,
  - при необходимости «зачтено как ...»,
  - время ответа.

### Экран прогресса

Добавляется карточка `⏱ Скорость ответа` со статистикой:

- среднее время;
- медиана;
- последние 10 ответов;
- размер журнала;
- доля быстрых ответов (`≤10 с`);
- точность;
- счётчики `rush` / `diag`;
- 3 самые «медленные» темы по среднему времени.

## Контент, добавленный в wave87x

Русский язык получил starter-cloze темы:

- `cloze8w87x`
- `cloze9w87x`
- `cloze10w87x`
- `cloze11w87x`

Grades 8/9/11 патчатся прямо в `grade*_data.js`, grade 10 — через lazy chunk `grade10_subject_rus_wave86s.js` и существующий `__wave86sApplyGrade10Subject(...)`.

## Что intentionally не делалось

- не добавлялся новый page-level CSS bundle — стиль инжектируется runtime-ом;
- не переписывался `engine10`;
- не менялись банки английского языка вручную, потому что в них уже много `___`-вопросов и runtime может подхватить их автоматически.

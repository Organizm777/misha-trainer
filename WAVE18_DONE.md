# Wave 18 — critical cleanup pack

## Что сделано

### 1) Развязаны облако и рейтинг `Молнии`
- добавлен новый слой `wave18_cleanup.js`
- `trainer_private` теперь трактуется только как настройка облачной синхронизации и кода восстановления
- добавлена отдельная настройка `trainer_rush_public_<GRADE>` для публикации рекордов `Молнии`
- сохранена мягкая миграция: если отдельная настройка ещё не была задана, старое поведение наследуется от `trainer_private`

### 2) Локальный рейтинг стал явным fallback для всех классов
- modal `showRushRecords()` переписан поверх patch-layer
- в классах без `RUSH_BIN_ID` теперь явно показывается, что доступен локальный рейтинг на этом устройстве
- в классах с `RUSH_BIN_ID` показываются отдельно:
  - локальные рекорды на устройстве
  - общий рейтинг
- в modal добавлен явный toggle публикации в общий рейтинг

### 3) `pushRushRecord()` больше не зависит от приватного режима облака
- отправка рекорда в облако теперь проверяет:
  - есть ли `RUSH_BIN_ID`
  - включена ли публикация в общий рейтинг
- при сетевой ошибке запись по-прежнему уходит в `rush_queue`

### 4) UI-копия стала честнее
- `renderPrivacyBtn()` переписан: теперь это именно кнопка про облако, а не «скрыты/публикуются результаты» вообще
- статические подсказки в `Справке` для страниц классов патчатся на лету:
  - для классов без облака — локальный рейтинг + резервная копия
  - для классов с облаком — публикация рейтинга отдельно от облачного backup

### 5) Проверки усилены
- добавлен `rush_cleanup_audit.js`
- добавлены артефакты:
  - `RUSH_CLEANUP_AUDIT.md`
  - `RUSH_CLEANUP_AUDIT.json`
- `browser_e2e_smoke.py` расширен сценариями:
  - local-only rush fallback (`grade1`)
  - cloud/local split for rush (`grade10`)
- `validate_release.py` получил:
  - `check_wave18_hooks()`
  - `check_rush_cleanup_audit()`
  - `check_rush_cleanup_guards()`

## Что сознательно НЕ делалось в этой волне
- не трогал ещё архитектурный split `grade10` по банкам
- не делал полноценный cleanup `removeEventListener / clearTimeout` на весь engine
- не переводил `tests.html` в unit-test harness: страница сейчас живая и linked из `index.html`, поэтому старый пункт про её удаление считаю устаревшим для текущей кодовой базы
- не трогал accessibility/mobile блоки: они всё ещё впереди отдельными волнами

## Итог
Wave 18 закрывает один из самых неприятных product gaps: теперь cloud backup и rush leaderboard — разные вещи, а классы без облака больше не выглядят сломанными.
